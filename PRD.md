# Morpheus — 产品需求文档 (PRD)

## 1. 产品概述

**Morpheus** 是一个轻量化的梦境记录 Web 应用。用户醒来后通过语音快速记录梦境内容，AI 自动将零散的口述整理为结构化的故事摘要，并保存到 Google Sheets 中。

### 目标用户
- 经常做梦且希望记录梦境内容的人
- 对梦境中的创意素材（写作、艺术、游戏设计）感兴趣的人
- 希望追踪自己梦境模式的用户

### 核心价值
- **轻**：刚醒时脑子糊，语音一键记录，不需要打字
- **智**：AI 自动提炼故事摘要和标签，无需手动整理
- **连**：通过 Google Sheets 存储，数据属于用户自己
- **静**：整体体验安静、克制、可信，不做“高饱和 AI 工具感”

## 3. 数据结构

### 3.1 记录格式

每条梦境记录包含以下字段：

```json
{
  "date": "2026-05-09",
  "tags": {
    "mood": "恐怖/惊悚",
    "themes": ["恐怖", "密室逃脱", "轮回"],
    "elements": ["蟑螂", "蟑螂鼠", "狭窄通道"]
  },
  "summary": "一个恐怖版的密室逃脱游戏。蟑螂群聚成老鼠的形状，诡异而令人不安。在第四关时独自绕圈出不去，最终决定倒着往回走，爬过狭窄通道回到起点。",
  "raw": "昨晚做梦又梦见素材了...像密室逃脱真恐怖版本。那个密室逃脱那个环境真的非常恶劣，里面有蟑螂，各种乱七八糟的一些脏东西。"
}
```

| 字段 | 类型 | 说明 | 来源 |
|------|------|------|------|
| `date` | string (YYYY-MM-DD) | 记录日期 | 系统自动生成（当天） |
| `tags.mood` | string | 情绪/氛围（恐惧、温暖、焦虑、奇幻等） | AI 提取 |
| `tags.themes` | string[] | 主题标签 | AI 提取 |
| `tags.elements` | string[] | 元素标签 | AI 提取 |
| `summary` | string | AI 润色后的故事摘要，1-2 句话，有故事感 | AI 生成 |
| `raw` | string | 语音识别的原始文字 | 语音识别直接输出 |

### 3.2 Google Sheets 表结构

| 列名 | 对应字段 | 说明 |
|------|----------|------|
| A | Date | 日期 |
| B | Mood | 情绪 |
| C | Themes | 主题（逗号分隔） |
| D | Elements | 元素（逗号分隔） |
| E | Summary | 故事摘要 |
| F | Raw | 原始记录 |

## 4. 用户流程

### 4.1 首次使用

```text
打开网页
→ 看到 Morpheus logo、标题文案与大圆录音按钮（无需先绑定 Google）
→ 用户可直接开始录音
→ 当用户首次点击「保存记录」时，如未绑定 Google，则弹出授权引导
→ 跳转 Google 授权页面
→ 用户同意授权
→ 返回应用，自动创建「Morpheus Dreams」Spreadsheet
→ 显示「Google 账号已绑定」
→ 返回确认记录页，继续保存
```

### 4.2 记录梦境

```text
点击录音按钮
→ 进入录音中页面（波形动画 + 计时器 + 停止按钮）
→ 用户口述梦境内容
→ 录音结束条件：
   a. 用户点击停止按钮
   b. 达到 3 分钟上限自动停止并提示
→ 进入「AI 解析中」页面
→ 进入「确认记录」页面，显示：
   - 日期（自动填入当天，可修改）
   - 心情 mood（单选）
   - 主题/元素标签（支持增删）
   - 故事摘要（可编辑）
   - 原始记录（可编辑）
→ 点击「保存记录」
→ 如未绑定 Google，则先进入授权流程
→ 保存成功后进入成功页
→ 点击「查看故事集」进入故事集
→ 或点击「返回首页」继续记录
```

### 4.3 查看故事集

```text
点击底部「故事集」
→ 优先读取本地缓存（localStorage），秒开列表
→ 后台静默调用 Google Sheets API 检查是否有新数据
→ 如有更新则刷新列表并同步本地缓存
→ 显示梦境记录列表（按日期倒序）
→ 每条记录显示：日期 + 摘要 + 标签
→ 点击某条记录，进入「故事详情页」
→ 首次使用显示空状态引导
```

缓存策略：
- 每次成功保存时同步写入 localStorage 一份副本
- 打开故事集时先展示本地数据，再后台同步
- 如本地缓存为空且已绑定 Google，则从 Sheets 全量拉取一次

### 4.4 AI 解析异常处理

```text
录音结束
→ 语音识别返回文字
→ 如果识别文字为空或极短（< 5 字）：
   提示「录音内容太少，请重新记录」
→ 如果 AI 解析超时（> 10 秒）或返回格式错误：
   提示「AI 解析失败」
   提供两个选项：
   a. 重试
   b. 手动填写
→ 手动填写模式下仅预填 raw 字段，其余字段用户手动补充
```

### 4.5 网络异常处理

```text
离线状态：
→ 首页顶部出现提示条「需要网络连接才能使用语音识别」

AI 解析或保存时断网：
→ 显示「网络连接失败」
→ 提供「重试」按钮
→ 已解析数据暂存在内存中，不丢失

Google Sheets 写入失败：
→ 提示「保存失败，请检查网络或 Google 账号授权状态」
→ 提供「重试保存」与「复制内容」两个操作
```

## 5. 页面设计

### 5.1 页面清单

| 页面 | 说明 |
|------|------|
| 1. 首页 · 记录 | Morpheus logo + 标题文案 + 大圆录音按钮 + 底部导航 |
| 2. 录音中 | 波形动画 + 计时器 + 停止按钮 |
| 3. AI 解析中 | 细密颗粒感加载动画 + 提示文案 |
| 4. 确认记录 | 心情、标签、故事摘要、原始记录、操作按钮 |
| 5. 保存成功 | 成功图形 + 「查看故事集」「返回首页」 |
| 6. 故事集 | 搜索框 + 列表 + 空状态 |
| 7. 故事详情 | 完整查看单条梦境内容 |
| 8. 设置 | Google 绑定、Sheet 状态、关于 |

### 5.2 底部导航

| Tab | 图标 | 说明 |
|-----|------|------|
| 记录 | 麦克风 | 返回首页/录音 |
| 故事集 | 书本/书页 | 查看梦境记录列表 |

### 5.3 视觉风格（重要）

本项目 UI 采用 **Aesop 风格的数字产品转译**，而不是科技 SaaS 风格：

- 基调：安静、克制、温暖、可信
- 关键词：**极简 / 质感 / 呼吸感 / 生活方式品牌感**
- 禁止使用高饱和渐变紫、荧光蓝、厚重玻璃拟态、夸张 AI 发光效果
- 使用大量留白、细线条、柔和边框、低对比暖色面
- 视觉重点放在内容本身，而不是特效

详见 `UI_SPEC_AESOP.md`

## 6. 技术架构

### 6.1 前端
- 技术栈：纯 HTML + CSS + JavaScript（无框架）
- 部署：Cloudflare Pages
- 目标：移动端优先，桌面端自适应

### 6.2 语音识别
- 方案：Web Speech API（`SpeechRecognition`）
- 语言：中文（`zh-CN`）
- 模式：点击开始录音 → 点击停止录音
- 限制：最长 3 分钟
- 降级：不支持时展示手动输入框

### 6.3 AI 解析
- 模型：待接入
- 方式：通过 Cloudflare Worker 中转
- 返回：严格 JSON
- 超时：10 秒
- 容错：
  - 输入过短直接跳过
  - 非法 JSON 自动重试 1 次
  - 失败则进入手动填写模式
  - 前端做 schema 校验与默认值兜底

### 6.4 存储
- 方案：Google Sheets API v4
- 认证：Google OAuth 2.0
- 自动创建：`Morpheus Dreams`
- 表头：Date / Mood / Themes / Elements / Summary / Raw
- 本地存储：OAuth token、Sheet ID、缓存记录
- 如果 Sheet 被删除，提示并允许重建

### 6.5 OAuth
- 模式：Testing
- 范围：
  - `https://www.googleapis.com/auth/spreadsheets`
  - `https://www.googleapis.com/auth/drive.file`

## 7. 非功能需求

### 7.1 性能
- 首页加载时间 < 2 秒
- AI 解析响应时间 < 5 秒（理想）
- 页面切换无明显卡顿

### 7.2 兼容性
- 浏览器：Chrome（推荐）、Safari
- Firefox：展示手动输入降级提示
- 屏幕：支持 320px 及以上宽度

### 7.3 隐私
- 梦境记录存储在用户自己的 Google Sheets 中
- 音频不经过本应用服务器
- AI 解析仅发送文字，不发送音频
- 不收集额外个人信息

### 7.4 安全
- AI API Key 不暴露前端
- v0.1 可接受 localStorage 存 token
- 后续可迁移到更安全的 session 方案
- 所有请求强制 HTTPS
